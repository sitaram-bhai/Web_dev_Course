Core Internet Technology

An IP (Internet Protocol) address is a unique numerical identifier assigned to every device connected to the internet. IP addresses are used to identify and communicate with devices on the internet, just like phone numbers are used to identify and communicate with people on a phone network. There are two versions of IP addresses in use today: IPv4 and IPv6.

IPv4 IPv6
32-bit address length, expressed in decimal numbers separated by periods. 128-bit address length, expressed in hexadecimal numbers separated by colons.
Supports approximately 4.3 billion unique addresses. Supports an astronomically large number of unique addresses (approximately 3.4 x 10^38), which should be sufficient for the foreseeable future.
The address space is divided into classes, with Class A, B, and C being the most common. The address space is flat, with no distinction between classes.
Uses ARP (Address Resolution Protocol) to map IP addresses to MAC addresses. Uses Neighbor Discovery Protocol (NDP) to map IP addresses to MAC addresses.
Includes optional fields in the header, such as flags and checksum. Simplifies the header by removing optional fields and incorporating optional features as extensionheaders.
Uses broadcast to communicate with all devices on a network. Uses multicast to communicate with groups of devices on a network.

IP addresses are essential for routing data packets between devices on the internet. When a device sends a request to another device on the internet, it uses the destination device's IP address to route the data packet to the correct destination. IP addresses can be assigned dynamically or statically. Dynamic IP addresses are assigned automatically by a DHCP (Dynamic Host Configuration Protocol) server, while static IP addresses are manually assigned to a device and do not change unless manually reconfigured. In summary, IP addresses are unique numerical identifiers that are essential for communication and data transfer on the internet.

Data transmission

Data transmission refers to the process of sending digital data over a communication channel or network from one device to another. Here are the basic steps involved in data transmission:
  • Encoding: The digital data is converted into a format that can be transmitted over the communication channel, such as binary or hexadecimal.
  • Transmission: The encoded data is transmitted over the communication channel, which can be a wired or wireless connection.
  • Reception: The receiving device receives the transmitted data.
  • Decoding: The received data is decoded back into its original format, such as text or an image.

    • Data transmission can occur over various types of communication channels, such as copper wires, fiber optic cables, or wireless radio waves. The speed and quality of data transmission depend on several factors, such as the bandwidth of the communication channel, the distance between the transmitting and receiving devices, and the level of interference in the communication channel. Different protocols, such as TCP/IP, are used to manage the transmission of data over networks and ensure that the data is delivered reliably and efficiently.
